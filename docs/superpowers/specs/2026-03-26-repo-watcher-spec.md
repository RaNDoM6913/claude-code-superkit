## Проблема

Слежу за несколькими репо (oh-my-claudecode, get-shit-done, everything-claude-code и др.), чтобы вовремя заметить полезные фичи для superkit. Ручная проверка — неэффективно и забывается.

## Цель

Автоматическая ежедневная проверка целевых репо с AI-анализом: что нового, стоит ли нам это внедрять.

## Варианты реализации

### Вариант A: Claude Code Scheduled Trigger (cron)

Claude Code поддерживает `schedule` skill — запуск remote agent по cron.

```
/schedule create --name repo-watcher --cron "0 9 * * *" --prompt "..."
```

**Плюсы:** нативно в Claude Code, AI-анализ из коробки
**Минусы:** требует API credits на каждый запуск, зависит от remote agent availability

### Вариант B: GitHub Actions Workflow

`.github/workflows/repo-watcher.yml` — cron workflow, проверяет репо через `gh`, создаёт Issue с отчётом.

```yaml
on:
  schedule:
    - cron: '0 9 * * *'  # каждый день в 9:00

jobs:
  watch:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Check repos for updates
        run: bash scripts/repo-watcher.sh
      - name: Create issue if updates found
        if: env.HAS_UPDATES == 'true'
        run: gh issue create --title "..." --body "$REPORT"
```

**Плюсы:** бесплатно, надёжно, создаёт Issues в нашем репо
**Минусы:** нет AI-анализа (только список изменений), нужен PAT

### Вариант C: Shell скрипт + SessionStart хук

Хук при старте сессии Claude Code проверяет "прошло ли 24ч с последней проверки?" — если да, запускает проверку.

```bash
# .claude/scripts/hooks/repo-watcher-session.sh
LAST_CHECK_FILE="$HOME/.claude/repo-watcher-last-check"
NOW=$(date +%s)
LAST=$(cat "$LAST_CHECK_FILE" 2>/dev/null || echo "0")
DIFF=$((NOW - LAST))

if [ "$DIFF" -gt 86400 ]; then  # 24h
  echo "$NOW" > "$LAST_CHECK_FILE"
  # inject prompt to check repos
fi
```

**Плюсы:** работает при каждой сессии, AI анализ в Claude Code
**Минусы:** не проверяет если не открываешь Claude Code

### Вариант D: Гибрид — GitHub Actions + Claude Code анализ

1. GitHub Actions раз в сутки собирает diff (releases, commits) → сохраняет в файл
2. SessionStart хук подгружает этот файл в контекст
3. Claude Code анализирует и предлагает действия

**Плюсы:** лучшее из обоих миров
**Минусы:** сложнее настроить

## Мой рекомендуемый вариант: B + элементы D

**GitHub Actions** как основа (надёжный cron), с shell-скриптом который:
1. Для каждого репо: `gh release list --limit 1` + `gh api repos/.../commits?since=YESTERDAY`
2. Сравнивает с сохранённым состоянием (`.github/repo-watcher-state.json`)
3. Если есть новое — создаёт GitHub Issue в нашем репо с отчётом
4. Issue заполняется структурированно: что нового, файлы, потенциальный интерес

Потом при открытии Claude Code — просто говоришь "проверь открытые issues от repo-watcher" и получаешь AI-анализ.

## Данные для отслеживания

Для каждого репо собирать:
- **Последний release** (tag, дата, release notes)
- **Новые коммиты** за последние 24ч (title, files changed)
- **Новые/изменённые файлы** в ключевых директориях (agents/, skills/, hooks/)
- **Изменения в README** (новые фичи в описании)
